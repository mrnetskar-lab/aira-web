AIRA PATCH — CAMERA BUTTON + IMAGE GENERATION FLOW
====================================================

Goal
----
Add a camera button to chat that lets the user generate an image from the current scene.

Scope
-----
- client/index.html
- client/style.css
- client/app.js
- server/routes/ai.js

No engine rewrites.
No Messages changes.
Keep this as a simple first version.

What this version does
----------------------
- adds a camera button beside Send in the main chat composer
- opens a small prompt flow
- sends a request to the server
- server returns an image URL (or placeholder if not wired yet)
- image appears inside the chat as a system image card

IMPORTANT
---------
This patch assumes you already have:
- main chat working
- /api/ai/run working
- current AIRA shell structure

If you do NOT have image generation wired yet, this patch still builds the UI and request flow.

STEP 7 — Add generateChatImage() helper

Add this full function below sendMessage() or near the other async chat helpers:

  async function generateChatImage() {
    const promptBase = elements.chatInput.value.trim();
    const prompt = window.prompt(
      "Describe the image you want to generate.",
      promptBase || "A cinematic scene inside the AIRA world"
    );

    if (!prompt) return;

    const placeholderTime = formatTime();

    state.messages.push({
      type: "system",
      text: "AIRA is framing the scene...",
      time: placeholderTime,
    });
    renderMessages();

    elements.cameraBtn.disabled = true;

    try {
      const response = await fetch("/api/ai/generate-image", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          prompt,
          context: {
            view: state.currentView,
            activeApp: state.activeApp,
          },
        }),
      });

      const payload = await response.json();

      if (!response.ok || !payload?.ok || !payload?.imageUrl) {
        throw new Error(payload?.error || "Image generation failed");
      }

      state.messages.push({
        type: "image",
        url: payload.imageUrl,
        alt: payload.alt || prompt,
        caption: payload.caption || prompt,
        time: formatTime(),
      });

      renderMessages();
    } catch (error) {
      console.error(error);
      state.messages.push({
        type: "system",
        text: "The image did not resolve. Try again.",
        time: formatTime(),
      });
      renderMessages();
    } finally {
      elements.cameraBtn.disabled = false;
    }
  }

STEP 8 — Bind the camera button

In bindEvents(), add:

  elements.cameraBtn.addEventListener("click", async () => {
    await generateChatImage();

PART 4 — SERVER ROUTE
==================================================

STEP 9 — Add a temporary image generation endpoint

In server/routes/ai.js, add a new route:

  router.post("/generate-image", async (req, res) => {
    try {
      const prompt = String(req.body?.prompt || "").trim();

      if (!prompt) {
        return res.status(400).json({
          ok: false,
          error: "Missing prompt",
        });
      }

      // TEMPORARY VERSION:
      // Replace this with your real image generation provider later.
      // For now return a deterministic placeholder image.

      const encoded = encodeURIComponent(prompt);
      const imageUrl = `https://placehold.co/1024x1024/png?text=${encoded}`;

      return res.json({
        ok: true,
        imageUrl,
        alt: prompt,
        caption: prompt,
      });
    } catch (error) {
      console.error(error);
      return res.status(500).json({
        ok: false,
        error: "Image generation route failed",
      });
    }
  });
