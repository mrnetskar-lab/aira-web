2) ADD DECISION METHOD
======================================================================

TARGET:
`src/engine/systems/RelationshipContinuitySystem.js`

ADD METHOD:

```js
decideResponseDisposition({ state, character, input }) {}
```

PURPOSE
Return one of:
- `reply_now`
- `reply_brief`
- `reply_later`
- `no_reply`

SUGGESTED IMPLEMENTATION:

```js
decideResponseDisposition({ state, character, input }) {
  const availability = state.continuity?.availability?.[character];
  const rel = state.relationships?.[character] || {};

  if (!availability) {
    return { mode: 'reply_now', reason: null };
  }

  if (availability.sleeping) {
    availability.replying = false;
    availability.willReplyLater = false;
    availability.noReplyReason = 'sleeping';
    return { mode: 'no_reply', reason: 'sleeping' };
  }

  const trust = normalize01(rel.trust, 0.5);
  const attraction = normalize01(rel.attraction, 0.3);
  const attachment = normalize01(rel.attachment, 0.3);
  const resentment = normalize01(rel.resentment, 0);
  const pride = normalize01(rel.pride, 0.3);

  const warmth = trust * 0.35 + attraction * 0.25 + attachment * 0.20 - resentment * 0.10 - pride * 0.10;
  const lower = String(input || '').toLowerCase();

  const vulnerable =
    /\b(sorry|miss you|need you|want you|love|trust|hurt|stay)\b/.test(lower);

  if (availability.busy) {
    if (Math.random() < 0.45) {
      availability.replying = true;
      availability.willReplyLater = false;
      availability.noReplyReason = null;
      return { mode: 'reply_brief', reason: 'busy' };
    }

    if (Math.random() < 0.65) {
      availability.replying = false;
      availability.willReplyLater = true;
      availability.noReplyReason = 'busy';
      return { mode: 'reply_later', reason: 'busy' };
    }

    availability.replying = false;
    availability.willReplyLater = false;
    availability.noReplyReason = 'busy';
    return { mode: 'no_reply', reason: 'busy' };
  }

  if (resentment > 0.55 && pride > 0.55 && Math.random() < 0.45) {
    availability.replying = false;
    availability.willReplyLater = false;
    availability.noReplyReason = 'pride';
    return { mode: 'no_reply', reason: 'pride' };
  }

  if (vulnerable && warmth > 0.42 && Math.random() < 0.30) {
    availability.replying = false;
    availability.willReplyLater = true;
    availability.noReplyReason = 'uncertain';
    return { mode: 'reply_later', reason: 'uncertain' };
  }

  if (Math.random() < 0.14) {
    availability.replying = false;
    availability.willReplyLater = false;
    availability.noReplyReason = 'distance';
    return { mode: 'no_reply', reason: 'distance' };
  }

  availability.replying = true;
  availability.willReplyLater = false;
  availability.noReplyReason = null;
  return { mode: 'reply_now', reason: null };
}
```

IMPORTANT
This should stay probabilistic and light.
Do not overfit it.

3) ADD LATER-QUEUE HOOKING
======================================================================

TARGET:
`src/engine/systems/RelationshipContinuitySystem.js`

ADD METHOD:

```js
maybeQueueDispositionFollowUp({ state, character, disposition, input }) {}
```

PURPOSE
If disposition is `reply_later`, queue a later event.

SUGGESTED IMPLEMENTATION:

```js
maybeQueueDispositionFollowUp({ state, character, disposition, input }) {
  if (!disposition || disposition.mode !== 'reply_later') return;

  const delayTurns =
    disposition.reason === 'busy'
      ? 1 + Math.floor(Math.random() * 2)
      : 2 + Math.floor(Math.random() * 2);

  const type =
    disposition.reason === 'busy'
      ? 'busy_reply'
      : 'soft_check_in';

  this.queueFollowUp({
    state,
    character,
    type,
    delayTurns,
    reason: disposition.reason || 'continuity'
  });
}
```

4) PRIMARY RESPONSE SUPPRESSION
======================================================================

TARGET:
`src/engine/brain/AiraBrainController.js`

PURPOSE
Not every chosen character should always answer immediately.

ADD CONTINUITY DEPENDENCY
Make sure controller can access:
```js
this.continuity
```
either by constructor injection or by passing through the engine wiring pattern already used elsewhere.

BEFORE generating the primary response, do:

```js
const disposition = this.continuity?.decideResponseDisposition({
  state,
  character: primary.name,
  input
}) || { mode: 'reply_now', reason: null };

this.continuity?.maybeQueueDispositionFollowUp({
  state,
  character: primary.name,
  disposition,
  input
});
```

THEN APPLY:

### Case 1 — `reply_now`
Proceed normally.

### Case 2 — `reply_brief`
Proceed, but override `responseMode` to `brief` for this character.

### Case 3 — `reply_later`
Skip immediate spoken response for this character.
Allow another character to answer if available.
If no other character answers, return no immediate line and rely on queued event later.

### Case 4 — `no_reply`
Skip immediate spoken response for this character.
Do not queue anything unless another system later chooses to.

IMPORTANT
No-reply is valid behavior.
Do not force a filler line.

5) SECONDARY RESPONSE RULE
======================================================================

TARGET:
`src/engine/brain/AiraBrainController.js`

If primary disposition is `reply_later` or `no_reply`,
do NOT let the room go dead every time.

Rule:
- try one secondary responder if one exists
- especially in group chat mode
- if none qualifies, allow silence

This keeps realism without making the app feel broken.
