So the current bug is primarily an API / engine / UI contract mismatch.

4) PATCH C â€” FIX ROUTE ASYNC / RESPONSE SAFETY
======================================================================

TARGET
`server/routes/ai.js`

PROBLEM
Your current frontend is doing:
```js
const payload = await response.json();
const interference = payload.result?.interference || {};
```
So `/api/ai/run` must reliably return the final engine result shape.

The older pack routes shown in the uploaded packs call:
```js
const result = engine.orchestrator.run(input);
```
without `await`, which is wrong for the richer async engine path.

PATCH INTENT
Make sure the live route does all of the following:
- uses `await engine.orchestrator.run(input)`
- returns `result` exactly as orchestrator produced it
- returns `state`, `issues`, `patches`
- keeps existing HTTP shape

REQUIRED ROUTE SHAPE
```js
router.post('/run', async (req, res) => {
  try {
    const input = String(req.body?.input || '').trim();

    if (!input) {
      return res.status(400).json({
        ok: false,
        error: 'Input is required.'
      });
    }

    const result = await engine.orchestrator.run(input);

    return res.json({
      ok: true,
      result,
      state: engine.orchestrator.getState(),
      recentMemory: engine.memory.getRecent(12),
      issues: engine.observer.getIssues(),
      patches: engine.patchWriter.getPending()
    });
  } catch (error) {
    console.error('AI route error:', error);
    return res.status(500).json({
      ok: false,
      error: error.message || 'Internal server error.'
    });
  }
});
```

WHY
Without this, the frontend may receive stale or incomplete result state.

6) PATCH E â€” FIX CURRENT FRONTEND BUGS IN app.js
======================================================================

TARGET
`client/app.js`

IMPORTANT
This file is Layout-owned under the contract, but these are integration fixes needed for the current UI to work with the AI contract. The frontend already contains the newer hooks.

BUG 1 â€” USER MESSAGE IS RENDERED BEFORE FINAL INTERFERENCE STATE
CURRENT BEHAVIOR
The app pushes a normal user message first, renders it, then later mutates it with `setTimeout()` if interference is active.

WHY IT FEELS WRONG
That creates an â€œedited after sendâ€ feel instead of â€œthis is what got sent.â€

FIX
Move the user push until AFTER the payload is known.

REPLACE THE CURRENT FLOW INSIDE `sendMessage(text)` WITH:

```js
async function sendMessage(text) {
  if (state.isSending) return;

  state.isSending = true;
  elements.sendBtn.disabled = true;

  try {
    const response = await fetch("/api/ai/run", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ input: text }),
    });

    const payload = await response.json();

    if (!response.ok || !payload?.ok) {
      throw new Error(payload?.error || "Request failed");
    }

    state.latestState = payload.state ?? null;
    updatePresenceHooks();

    const interference = payload?.result?.interference ?? null;
    const time = formatTime();

    const displayText = interference?.active
      ? (interference.perceivedInput || text)
      : text;

    state.messages.push({
      type: "user",
      text: displayText,
      anomalous: Boolean(interference?.active),
      bubbleClass: interference?.uiEffect?.bubbleClass || "",
      time,
    });

    if (interference?.whisper) {
      const whisperDelay = (interference.uiEffect?.delayMs ?? 120) + 200 + Math.random() * 500;
      setTimeout(() => {
        state.messages.push({
          type: "system",
          text: interference.whisper,
          whisper: true,
          time: formatTime(),
        });
        renderMessages();
      }, whisperDelay);
    }

    state.messages.push(...normalizeResponses(payload));

    if (Array.isArray(payload.issues) && payload.issues.length && window.__AIRA_DEV_MODE__) {
      state.messages.push({
        type: "system",
        text: `Issues: ${payload.issues.map((issue) => issue.type || issue).join(", ")}`,
        time: formatTime(),
      });
    }

    renderMessages();
  } catch (error) {
    console.error(error);
    state.messages.push({
      type: "system",
      text: "Something slipped in the dark. Try again.",
      time: formatTime(),
    });
    renderMessages();
  } finally {
    state.isSending = false;
    elements.sendBtn.disabled = false;
  }
}
```

WHY
- avoids awkward mutation
- keeps the sent message truthful to the final system output
- keeps interference creepy instead of buggy

BUG 2 â€” WHISPERS ARE NOT VISUALLY DISTINCT
CURRENT BEHAVIOR
Whispers are pushed as plain `type: "system"` entries with no flag.

FIX
Render them differently.

IN `renderMessages()`, CHANGE:
```js
<div class="system-note">${escapeHtml(message.text)}</div>
```

TO:
```js
<div class="system-note${message.whisper ? " system-note--whisper" : ""}">
  ${escapeHtml(message.text)}
</div>
```

WHY
Whispers need their own visual lane.

BUG 3 â€” DEV ISSUE RENDERING IS WRONG
CURRENT BEHAVIOR
The app currently does:
```js
text: `Issues: ${payload.issues.join(", ")}`
```
But `issues` is an array of objects in the current engine/dev pattern, not strings. This will print `[object Object]`.

FIX
Use:
```js
text: `Issues: ${payload.issues.map((issue) => issue.type || issue).join(", ")}`
```

BUG 4 â€” TIMESTAMP DRIFT ON DELAYED MUTATION
By mutating the original user message after a timeout, the UI can show a visually shifted message with the old timestamp and a delayed rewrite. Removing mutation fixes this automatically.

7) PATCH F â€” FIX NORMALIZE RESPONSES SAFETY
======================================================================

TARGET
`client/app.js`

PATCH INTENT
Make response parsing resilient even if `meta` is missing.

REPLACE `normalizeResponses(payload)` WITH:

```js
function normalizeResponses(payload) {
  const responses = payload?.result?.responses ?? [];
  const gmLine = payload?.result?.gmLine ?? null;
  const time = formatTime();

  const items = responses.map((entry) => ({
    type: "character",
    agent: entry.agent || "AIRA",
    text: entry.spoken || "",
    thought: entry.thought || "",
    anomalous: Boolean(entry.anomalous),
    toneClass: entry.meta?.toneClass || "",
    anomalyAware: Boolean(entry.meta?.anomalyAware),
    time,
  }));

  if (gmLine && window.__AIRA_DEV_MODE__) {
    items.push({
      type: "system",
      text: gmLine,
      time,
    });
  }

  return items;
}
```

WHY
This matches the current UI assumptions and fails safely.
